Ikube is a processing platform, multi threaded, grid based for extreme volumes of data, specifically for analytics, using statistical models and neural networks for analyses. Future ready for IoT when processing real-time analytics in extreme volumes becomes popular. Self organizing, auto network discovery, fully distributed and load balanced, models built in parallel for larger models. And combined enterprise analytics/search out of the box.

Serenity is a code coverage plugin for Jenkins with dynamic byte code injection using ASM, including static analysis for the major code quality metrics, abstractness, efferent and afferent coupling, stability, complexity and distance from main. Integrated in the library is a production profiler. This will instrument only methods, and typically increases the cpu consumption by about 3%. Note that it is not recommended for profiling transactional methods, as these methods will already have been instrumented, and doing so again could have interesting side effects.

Scanner is a network scanner, written in Scala(just for a laugh), that, well, scans a specific network range, and ports of course. Can be run from the command line, and as a library to be used internally from Java/Scala or probably Groovy too in fact.

The client gui is a search gui for the ikube server. Not finished, but could be interesting. Lots of work using Eclipse RCP, difficult and expensive but in the end worth it because it is quite attractive. Even programmers can create nice looking documents in Eclipse RCP, really.
